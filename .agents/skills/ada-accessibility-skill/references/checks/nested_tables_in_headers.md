# Nested Tables in Table Headers

Nesting a `<table>` element inside a `<th>` element breaks assistive technology's ability to communicate header-cell relationships. Screen reader users rely on table header associations to understand the structure and meaning of tabular data — a nested table inside a header destroys these relationships, making the data incomprehensible. This applies to HTML `<table>`/`<th>` elements and Salt Design System `<Table>`/`<TableContainer>`/`<TH>` components.

## Checkpoint Covered

- **1.3.1-28** — Table Headers Must Not Contain Nested Tables

## Common Issues

- HTML `<table>` element nested inside a `<th>` element
- Salt `<Table>` or `<TableContainer>` component nested inside a Salt `<TH>` component

## Remediation Rules

1. **If a `<th>` or Salt `<TH>` contains a `<table>`, `<Table>`, or `<TableContainer>` descendant**: This is always a violation. **Flag for manual review** — the developer must restructure the table layout. The agent cannot auto-fix this because extracting a nested table requires design-level decisions about how to present the data.
2. **Detection scope**: Scan all `<th>` and Salt `<TH>` elements. Check if any descendant (direct or nested) is a `<table>`, `<Table>`, or `<TableContainer>` element. This includes conditionally rendered content (e.g., ternary expressions, `&&` short-circuit) — if a table *can* render inside a header, flag it.
3. **No auto-fix**: Every detected violation is flagged for manual review. Do not attempt to restructure the table — this requires developer judgment about how to present the data without nesting.

### When to Flag for Manual Review

- Always. This check has no auto-fix. Every detected instance of a `<table>`/`<Table>`/`<TableContainer>` inside a `<th>`/`<TH>` must be flagged for manual review. Record the issue in a11y.md with `Status: flagged for review` and a brief `Why it's a defect` explanation. Do NOT write recommendation text or proposed solutions into a11y.md — specific remediation options are discussed during the interactive walkthrough only, where the agent performs a fresh contextual evaluation of the code.

### Important Notes

- **Salt components use PascalCase**: Salt's table components are `<Table>`, `<TableContainer>`, `<THead>`, `<TBody>`, `<TR>`, `<TH>`, and `<TD>`. These map directly to their HTML equivalents (`<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`).
- **Detection includes both direct and nested descendants**: A `<table>` nested several levels deep inside a `<th>` (e.g., inside a `<div>` inside the `<th>`) is still a violation.
- **This check does not apply to `<td>` elements**: Only `<th>` (table header) elements are checked. Nested tables inside `<td>` (table data) cells are a separate concern not covered by this checkpoint.
- **AG Grid and `role="grid"` elements are out of scope**: This check applies only to HTML `<table>`/`<th>` and Salt `<Table>`/`<TH>` elements. It does not apply to AG Grid (`<AgGridReact>`), elements with `role="grid"`, or other third-party data grid components whose internal table markup is generated by the library and not directly authored by the developer.

## Examples

### Using Salt Design System

```jsx
import { Table, TableContainer, TBody, TD, TH, THead, TR } from "@salt-ds/core";

// CORRECT: Table headers contain only text content
<TableContainer>
  <Table>
    <caption>Planet Data</caption>
    <THead>
      <TR>
        <TH>Planet</TH>
        <TH>Type</TH>
        <TH>Atmosphere</TH>
      </TR>
    </THead>
    <TBody>
      <TR>
        <TD>Earth</TD>
        <TD>Terrestrial</TD>
        <TD>Nitrogen, Oxygen</TD>
      </TR>
    </TBody>
  </Table>
</TableContainer>

// INCORRECT: Table nested inside a TH element — flag for manual review
<TableContainer>
  <Table>
    <THead>
      <TR>
        <TH>
          <Table>
            <THead>
              <TR>
                <TH>Sub Header A</TH>
                <TH>Sub Header B</TH>
              </TR>
            </THead>
          </Table>
        </TH>
        <TH>Other Column</TH>
      </TR>
    </THead>
    <TBody>
      <TR>
        <TD>Data</TD>
        <TD>Data</TD>
      </TR>
    </TBody>
  </Table>
</TableContainer>

// INCORRECT: TableContainer nested inside a TH element — flag for manual review
<TableContainer>
  <Table>
    <THead>
      <TR>
        <TH>
          <TableContainer>
            <Table>
              <THead>
                <TR>
                  <TH>Nested Header</TH>
                </TR>
              </THead>
            </Table>
          </TableContainer>
        </TH>
      </TR>
    </THead>
    <TBody>
      <TR>
        <TD>Data</TD>
      </TR>
    </TBody>
  </Table>
</TableContainer>
```

### HTML Implementation

```html
<!-- CORRECT: Table headers contain only text content -->
<table>
  <caption>Planet Data</caption>
  <thead>
    <tr>
      <th>Planet</th>
      <th>Type</th>
      <th>Atmosphere</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Earth</td>
      <td>Terrestrial</td>
      <td>Nitrogen, Oxygen</td>
    </tr>
  </tbody>
</table>

<!-- INCORRECT: Table nested inside a th element — flag for manual review -->
<table>
  <thead>
    <tr>
      <th>
        <table>
          <thead>
            <tr>
              <th>Sub Header A</th>
              <th>Sub Header B</th>
            </tr>
          </thead>
        </table>
      </th>
      <th>Other Column</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Data</td>
      <td>Data</td>
    </tr>
  </tbody>
</table>

<!-- INCORRECT: Table nested deeper inside a th via a wrapper div — still a violation -->
<table>
  <thead>
    <tr>
      <th>
        <div class="header-wrapper">
          <table>
            <thead>
              <tr>
                <th>Nested Header</th>
              </tr>
            </thead>
          </table>
        </div>
      </th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Data</td>
    </tr>
  </tbody>
</table>
```
