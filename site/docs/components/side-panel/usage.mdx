## Using the component

A side panel covers the entire height of the container it is nested within and is always non-modal. This means that the content beside it is simultaneously interactive when expanded.

Always set a logical width suitable for the content inside a side panel. Refer to Salt's Forms pattern for information on logical forms width, and the typography foundation for guidance on paragraph text width.

### When to use

- To provide a persistent, side-by-side workspace that supports ongoing parallel tasks, such as referencing details, inspecting data, filtering results, or editing attributes, while users continue their main task.
- To keep supporting information and controls visible and accessible, improving accuracy and efficiency by maintaining context.
- To allow users to view and interact with supplementary information without navigating away from their primary workflow.
- To facilitate workflows where maintaining visibility of both main content and supporting content is important for productivity.

### When not to use

- For a temporary, focused workspace that lets users handle a small, self-contained action. Instead, use Drawer.
- For a background that organizes content in an application. Instead, use Panel.

## Import

To import `SidePanel`, `SidePanelGroup` and `SidePanelTrigger` from the lab Salt package, use:

```js
import { SidePanel, SidePanelGroup, SidePanelTrigger } from "@salt-ds/lab";
```
