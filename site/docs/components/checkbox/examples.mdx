<LivePreview componentName="checkbox" exampleName="VerticalGroup" >

### Checkbox group: vertical (default)

A checkbox group allow selection of one or more values from a given set of choices. They can be aligned horizontally or vertically.

Guidance:

- Keep values short and as concise as possible in the horizontal orientation.
- Do not truncate checkbox values.
- The checkbox should always be placed left aligned to the value.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="HorizontalGroup" >

### Chekcbox group: horizontal

The checkbox group layout can be set to horizontal by setting `direction="horizontal"`.

Guidance:

- Keep values short and as concise as possible in the horizontal orientation.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="WrapGroup" >

### Wrapping checkbox group

The default behaviour for checkbox group in a horizontal alignment allows checkbox options to wrap onto the next line as the viewport size changes, if they don’t fit within a container.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="NoWrapGroup" >

### Value text wrapping

You can configure Checkbox groups to maintain a horizontal alignment but allow text label wrapping by setting the prop `wrap=false`.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="Indeterminate" >

### Indeterminate checkbox

When a checkbox value is neither selected nor deselected, it is indeterminate. This is a third, separate state. A Checkbox with the prop `indeterminate="true"` will display the checkbox with an indeterminate icon. The most common use case is a nested checkbox group which shows a hierarchy of selections. If a group contains a mix of selected and unselected children, the parent will show an indeterminate state.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="Disabled">

### Disabled checkbox

A disabled Checkbox is not interactive or focusable, as it is not relevant to the user’s current use case. A Checkbox can be disabled by setting the disabled prop to true. This will disable all functionality.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="Readonly">

### ReadOnly checkbox

A readonly Checkbox permits the user to only read or copy the value, but not edit the value or change the state of the Checkbox.

A Checkbox with the prop `readOnly="true"` will suppress all functionality along with displaying readonly styling.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="Error">

### Validation states: error

Error state can be indicated stylistically by setting the `validationStatus` prop to "error". Use this state when you want to alert the user of a critical issue that is related to the checkbox.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="Warning">

### Validation states: warning

Warning state can be indicated stylistically by setting the `validationStatus` prop to "warning". Use this state when you want to alert the user of an issue that is related to the checkbox.

Guidance:

- When used in a group, use `validationStatus` on the group rather than the individual controls. Any status provided to the group will take precedence over the statuses applied directly to the nested controls.
- Avoid using `validationStatus` when a Checkbox is disabled. If provided, disabled functionality and styling will take precedence over any validation status styling.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="LongTextGroup">

### Long value support

Checkbox supports long values. If a text value wraps to multiple lines, it align the text to the top of the checkbox.

</LivePreview>

<LivePreview componentName="checkbox" exampleName="WithFormField" >

### Form Field compatibility

Checkbox group can be wrapped in a Form Field when it’s displayed within a Form context. This provides functionality built into Form Field for increased accessibility.

For more information, refer to the [Form Field](https://www.saltdesignsystem.com/salt/patterns/forms) guidance.

</LivePreview>
