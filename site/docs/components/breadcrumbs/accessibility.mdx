## Best practices

- `BreadcrumbsNext` renders a `<nav>` landmark. Give it an accessible name with `aria-label` or `aria-labelledby`, and ensure the name is unique when more than one breadcrumb trail is present on a page.
- Separators are decorative and are hidden from assistive technologies with `aria-hidden`.

## Current page

The current breadcrumb renders as text rather than a link, with `aria-current="page"` so assistive technologies can identify the page represented by the trail.

`aria-current="page"` is derived from the `current` prop or the final item; activating a link does not move it. Update the trail from your route or page state after navigation so exactly one item represents the current page.

In collapsed trails, hidden levels without an `href` render as text and are skipped by disclosure focus movement, while the current page stays visible in the trail.

## Keyboard interactions

<KeyboardControls>
<KeyboardControl keyOrCombos="Tab">

Tab moves focus to each linked breadcrumb and any collapse trigger in document order. If the disclosure is open, Tab moves focus from the trigger into the revealed breadcrumb links. The current page label is not focusable unless you render custom focusable content.

</KeyboardControl>
<KeyboardControl keyOrCombos="Enter">

When focus is on a linked breadcrumb, Enter navigates to that breadcrumb's destination.

</KeyboardControl>
<KeyboardControl keyOrCombos="Enter / Space">

When focus is on a collapsed disclosure trigger, this action toggles the disclosure and focus remains on the trigger. Use Tab to move into the revealed breadcrumb links.

</KeyboardControl>
<KeyboardControl keyOrCombos="Escape">

When the collapsed disclosure is open, Escape closes it and returns focus to the disclosure trigger.

</KeyboardControl>
</KeyboardControls>
