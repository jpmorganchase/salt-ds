## What is a tooltip
A tooltip is a small pop up that displays on hover or focus that should only be used to provide extra (non-essential) information about a [form field](https://www.saltdesignsystem.com/salt/components/form-field/examples), a [link](https://www.saltdesignsystem.com/salt/components/link/examples), a [button](https://www.saltdesignsystem.com/salt/components/button/examples), or other focusable element.

## Best practices
*Tooltips must be triggered by both focus and hover events and remains on the screen as long as the trigger has the focus. The focus does not move to the tooltip.
*Since a tooltip doesn't receive focus, don't include interactive elements as the user can't access them.
*The expand/collapsed state is intentionally not exposed
*Keep tool tips to a minimum and only when they provide extra (non-essential) information for the user.


## Anti-Pattern
Never apply a tooltip to non-interactive elements. As a rule, non-operable elements shouldn't receive keyboard focus to avoid unnecessary navigation steps.
A common anti-pattern is to use an (i) icon as the tooltip anchor however this isn't recommended for the following reasons:

- Icons do not receive focus by default
- Adding `tabindex` to make the icon focusable introduces an additional tab stop

## Keyboard interactions

<KeyboardControls>
  <KeyboardControl keyOrCombos="Tab">

    - Tooltips do not receive focus, so Tab does not move focus to the tooltip. Instead Tab moves focus to tooltip trigger elements like a button or link.

  </KeyboardControl>
  <KeyboardControl keyOrCombos="Escape">

    - If the tooltip is open, pressing Escape dismisses the tooltip.

  </KeyboardControl>
</KeyboardControls>
