## Using the component

### When to use

- To organize and group sets of related controls, such as actions and commands together.
- To improve the discoverability of related actions.
- To provide a consistent, predictable location for related controls within a page, panel, or section.

### When not to use

- For placing actions that are not directly related to the current context or content.

- As primary navigation between major sections or pages. Instead, use [`Navigation item`](../navigation-item).
- To display only one control.
- To display a brand logo, navigation, and utility actions. Instead, use [`App header`](/salt/patterns/app-header).

## Content

- Follow the respective content guidance for all components used inside a toolbar.
- Always place controls inside `TooltrayNext`. Both composition models require it.
- For simple toolbars, place `TooltrayNext` directly inside `ToolbarNext`. Use the `align` prop to position each tray at the `start`, `center`, or `end`.
- For more structure, wrap `TooltrayNext` groups in `ToolbarContentNext` and set its `position` prop. Use this when a position needs multiple groups, dividers, custom spacing, or content-level props.
- Use one composition model per toolbar.

## Import

To import `Toolbar` from the lab Salt package, use:

```js
import { ToolbarContentNext, ToolbarNext, TooltrayNext } from "@salt-ds/lab";
```
