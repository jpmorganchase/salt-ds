## Appearance

Toolbar supports two appearances: `bordered` or `transparent`. Transparent toolbars remove the default inset padding, so their contents align with the toolbar container edges. The default appearance is `bordered`.

## Variants

Bordered toolbars are available in three variants: `primary`, `secondary`, and `tertiary`. The default variant is `primary`.

## Content position

Controls must be placed into `TooltrayNext` trays to participate in grouping and overflow behavior.

Toolbar supports two positioning models:

- Use `align` on `TooltrayNext` to place individual trays at the `start`, `center`, or `end` of the toolbar.
- Use `ToolbarContentNext` with the `position` prop to define a content section that holds one or more `TooltrayNext`.

## Overflow

When the toolbar width is reduced and there isn't enough space for every control, trays move into overflow. Each tray collapses as a whole, keeping its controls grouped together.

### Collapse order

Use `overflowPriority` to define a custom collapse order. Higher priority values collapse sooner. Tooltrays with higher `overflowPriority` values move into the overflow before those with lower values.

### Independent collapse

By default, `overflowMode` is `"independent"`, so each `TooltrayNext` collapses on its own. As the toolbar width is reduced, trays move into the overflow one at a time, following their `overflowPriority`.

<LivePreview componentName="toolbar" exampleName="IndependentCollapse" />

### Named overflows

By default, trays collapse into a single shared overflow. Assign a named `overflowGroup` to route trays into their own overflow, each with its own overflow trigger. Use `overflowLabel` when the overflow trigger label should differ from the group name.

#### Best practices

If your toolbar has more than one overflow, give each named overflow a distinct label so the overflow triggers are easy to tell apart.

<LivePreview componentName="toolbar" exampleName="NamedOverflows" />

### Grouped collapse

Within a named overflow, set `overflowMode` to `"grouped"` on its trays to collapse them together as a single unit, rather than one at a time. Use this when trays are interdependent and should always appear together.
