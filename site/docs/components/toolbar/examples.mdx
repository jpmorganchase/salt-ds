## Appearance

Toolbar supports two appearances: `bordered` or `transparent`. The `bordered` appearance is set by default. Transparent toolbars are flush with the edges of the toolbar container.

<LivePreview componentName="toolbar" exampleName="Appearance" />

## Variants

Bordered toolbars are available in three variants: `primary`, `secondary`, and `tertiary`. The `primary` variant is set by default.

<LivePreview componentName="toolbar" exampleName="Variants" />

## Content position

Use `ToolbarContentNext` with the `position` prop to set where its `TooltrayNext` children appear: `start`, `center`, or `end` of the toolbar. Each `ToolbarContentNext` can contain one or more `TooltrayNext` components and optional dividers.

For simple toolbars, place `TooltrayNext` children directly in `ToolbarNext` and use `align` as shorthand for the same positions. Use one composition model per toolbar.

<LivePreview componentName="toolbar" exampleName="ContentSections" />

## Dividers

Use a [`Divider`](../divider) to separate groups of controls within a toolbar. Alternatively, apply `spacing-300` between groups when visual separation is enough.

## Overflow

### Independent collapse

By default, `overflowMode` is set to `"independent"`, which enables controls to collapse from right to left, one tool tray at a time. When the toolbar's available width shrinks and the content no longer fits, controls move into an overflow menu.

<LivePreview componentName="toolbar" exampleName="IndependentCollapse" />

### Grouped collapse

When `overflowMode` is set to `"grouped"`, controls can collapse together when the toolbar can no longer display all components. Use this when controls are interdependent and need to remain together to work effectively.

<LivePreview componentName="toolbar" exampleName="GroupedCollapse" />

### Dynamic collapse

By default, controls move into one overflow. To create separate overflow triggers, use named `overflowGroup` values. Use `overflowLabel` when the trigger label should differ from the group name.

#### Best practices

If your toolbar has more than one overflow, give each named overflow a distinct label so the triggers are easy to tell apart.

<LivePreview componentName="toolbar" exampleName="DynamicCollapse" />

### Collapse order

Use `overflowPriority` to define a custom collapse order. Tool trays with higher `overflowPriority` values move into overflow before tool trays with lower values.
