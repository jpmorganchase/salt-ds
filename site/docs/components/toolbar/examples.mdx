## Appearance

Toolbar supports two appearances: `bordered` or `transparent`. Transparent toolbars are flush with the edges of the toolbar container. The default appearance is `bordered`.

<LivePreview componentName="toolbar" exampleName="Appearance" />

## Variants

Bordered toolbars are available in three variants: `primary`, `secondary`, and `tertiary`. The default variant is `primary`.

<LivePreview componentName="toolbar" exampleName="Variants" />

## Content position

Toolbar supports two positioning models. Use `align` on `TooltrayNext` to place individual trays at the `start`, `center`, or `end` of the toolbar. Use `ToolbarContentNext` with the `position` prop to define a content section that holds one or more `TooltrayNext` components and optional dividers.

<LivePreview componentName="toolbar" exampleName="ContentSections" />

## Dividers

Use a [`Divider`](../divider) to separate groups of controls within a toolbar. Alternatively, apply `spacing-300` between groups when visual separation is enough.

<LivePreview componentName="toolbar" exampleName="Dividers" />

## Overflow

### Independent collapse

By default, `overflowMode` is `"independent"`, which moves each `TooltrayNext` into an overflow individually. When the toolbar's available width shrinks and the content no longer fits, `TooltrayNext` components move into an overflow menu.

<LivePreview componentName="toolbar" exampleName="IndependentCollapse" />

### Grouped collapse

When `overflowMode` is set to `"grouped"`, controls in the same `TooltrayNext` move into overflow together. Use this when controls are interdependent and need to remain together to work effectively.

<LivePreview componentName="toolbar" exampleName="GroupedCollapse" />

### Dynamic collapse

By default, controls move into one overflow. To create separate overflow triggers, use named `overflowGroup` values. Use `overflowLabel` when the trigger label should differ from the group name.

#### Best practices

If your toolbar has more than one overflow, give each named overflow a distinct label so the triggers are easy to tell apart.

<LivePreview componentName="toolbar" exampleName="DynamicCollapse" />

### Collapse order

Use `overflowPriority` to define a custom collapse order. `TooltrayNext` components with higher `overflowPriority` values move into overflow before those with lower values.
