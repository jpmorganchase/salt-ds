## Using the component

Use `BreadcrumbsNext` with `BreadcrumbNext` items to show a user's current location within a hierarchy. The parent renders the navigation landmark, ordered list, separators, collapse controls, and current item behavior.

## Content

- Keep breadcrumb labels concise and descriptive. Labels should match the destination or current page closely enough to make sense out of context.
- Keep the visible trail concise. Aim for no more than four visible levels, and use `maxItems` to collapse lower priority levels in deeper hierarchies.
- Use contextual labels when they help users understand their current task, such as "Edit profile" instead of "Profile".
- Shorten or combine low value levels only when the resulting trail still explains the user's location.
- Do not make the current page a link. `BreadcrumbsNext` renders the current item as text when it is marked with `current`, or when it is the final item.

## Import

To import `BreadcrumbsNext` and related components from the lab Salt package, use:

```js
import {
  BreadcrumbNext,
  BreadcrumbNextLabel,
  BreadcrumbNextTrigger,
  BreadcrumbsNext,
} from "@salt-ds/lab";
```

If you need to customize the separator icon, also import `SemanticIconProvider` from the core Salt package:

```js
import { SemanticIconProvider } from "@salt-ds/core";
```

For composed breadcrumb item icons, import the icon from the icons Salt package:

```js
import { HomeIcon } from "@salt-ds/icons";
```

## Breadcrumb items

### Text items

Use text children for the common case. Previous hierarchy levels should provide `href` and render as Salt `Link` elements. The current level renders as text with `aria-current="page"` and is not linked by default.

```tsx
<BreadcrumbsNext aria-label="Breadcrumb">
  <BreadcrumbNext href="/accounts">Accounts</BreadcrumbNext>
  <BreadcrumbNext href="/accounts/asset-management">
    Asset management
  </BreadcrumbNext>
  <BreadcrumbNext>Portfolio</BreadcrumbNext>
</BreadcrumbsNext>
```

### Setting the current level

If no child explicitly sets `current`, `BreadcrumbsNext` treats the final `BreadcrumbNext` child as the current page. Set `current` on a child when the current level needs to be declared explicitly.

### Composed items

For advanced composition, such as adding a tooltip or an icon inside the trigger, compose `BreadcrumbNextTrigger` and `BreadcrumbNextLabel` as children. `BreadcrumbNextLabel` provides the visible and accessible label for composed breadcrumbs. Collapsed disclosure items render through the same trigger path, so custom trigger props, router renderers, icons, and tooltip wrappers are preserved:

```tsx
<BreadcrumbNext href="/accounts/asset-management">
  <Tooltip
    content="Asset management · Mandate: Discretionary · Region: EMEA"
    placement="top"
  >
    <BreadcrumbNextTrigger>
      <BreadcrumbNextLabel>Asset management</BreadcrumbNextLabel>
    </BreadcrumbNextTrigger>
  </Tooltip>
</BreadcrumbNext>
```

When providing custom element children, always include `BreadcrumbNextTrigger`; otherwise `href` is not consumed and the item will not render as a link.

```tsx
<BreadcrumbNext href="/accounts">
  <BreadcrumbNextTrigger>
    <HomeIcon aria-hidden />
    <BreadcrumbNextLabel>Accounts</BreadcrumbNextLabel>
  </BreadcrumbNextTrigger>
</BreadcrumbNext>
```

### Custom separator icon

To customize the separator icon between breadcrumb items, provide `LevelSeparatorIcon` through `SemanticIconProvider`. The default separator remains `ChevronRightIcon`.

```tsx
<SemanticIconProvider iconMap={{ LevelSeparatorIcon: CustomSeparatorIcon }}>
  <BreadcrumbsNext aria-label="Breadcrumb">
    <BreadcrumbNext href="/accounts">Accounts</BreadcrumbNext>
    <BreadcrumbNext>Portfolio</BreadcrumbNext>
  </BreadcrumbsNext>
</SemanticIconProvider>
```

## Routing libraries

Use `render` to integrate with routing libraries such as [React Router](https://reactrouter.com/) or [Next.js](https://nextjs.org/docs/app/api-reference/components/link).

### Shared trail renderer

Add `render` to `BreadcrumbsNext` when the same router link should be used for the whole trail. Visible breadcrumb links and links inside the collapsed disclosure use the same renderer.

```tsx
import type { ComponentPropsWithoutRef } from "react";
import { Link as RouterLink } from "react-router";

function renderRouterLink({
  href = "",
  ...props
}: ComponentPropsWithoutRef<"a">) {
  return <RouterLink {...props} to={href} />;
}

<BreadcrumbsNext aria-label="Breadcrumb" render={renderRouterLink}>
  <BreadcrumbNext href="/accounts">Accounts</BreadcrumbNext>
  <BreadcrumbNext href="/accounts/asset-management">
    Asset management
  </BreadcrumbNext>
  <BreadcrumbNext>Portfolio</BreadcrumbNext>
</BreadcrumbsNext>;
```

### Per-item renderer

Use `render` on `BreadcrumbNext` when one breadcrumb needs different routing behavior or link options. This item renderer takes precedence over the shared renderer from `BreadcrumbsNext`, including when the item is collapsed into the disclosure.
