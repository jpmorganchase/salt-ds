## Default

Use the default composition for a concise trail with linked previous levels and a text current level.

### Best practices

- We recommend keeping the number of breadcrumb levels concise to a maximum of four, including the first level if it's displayed. Content is more easily discoverable when navigating a relatively wide and flat information hierarchy within applications. This prevents users from feeling overwhelmed.
- Use breadcrumbs to complement the application's main navigation—never to replace it.
- Don't show breadcrumbs when there is only one level in a hierarchy.

## Wrapped

When the content exceeds the available width, breadcrumbs can wrap onto subsequent lines. The breadcrumb and its separator should be on the same line.

Set `wrap` to allow the trail to wrap onto subsequent lines.

## Router integration

Use `render` to adapt breadcrumb links to a routing library such as React Router or Next.js. The same renderer is used for visible links and links inside the collapsed disclosure.

<LivePreview componentName="breadcrumbs-next" exampleName="RouterIntegration" />

## Expansion

Alternatively if there's sufficient horizontal space, use an expansion button to reveal all levels. Showing the levels inline can make the hierarchy of the levels more visible.

Set `collapseMode="expand"` to reveal hidden breadcrumb levels inline.
