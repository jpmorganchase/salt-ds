### Radio Button group (vertical)

Radio Button groups allow users to select one or more values from a given set of choices. They can be aligned horizontally or vertically.

When `direction="vertical"`

Guidance:

- Keep values short and as concise as possible in the horizontal orientation.
- Do not truncate radio button values.
- The radio should always be placed left aligned to the value.

### Wrapping Radio Button group

The default behaviour for radio button groups in a horizontal alignment allows radio button options to wrap onto the next line as the viewport size changes, if they don’t fit within a container.

</LivePreview>

<LivePreview componentName="radio-button" exampleName="NoWrap" >

### Text wrapping

You can configure radio button groups to maintain a horizontal alignment but allow text label wrapping in response to viewport changes. To enable label wrapping, set the prop `wrap=false`.

</LivePreview>

### Disabled Radio Button

A disabled Radio Button is not interactive or focusable, as it is not relevant to the user’s current use case. You can disable a radio button by setting the `disabled` prop to true. This will disable all functionality.

</LivePreview>

<LivePreview componentName="radio-button" exampleName="Readonly" >

### ReadOnly Radio Button

A read-only Radio Button permits the user to only read or copy the value, but not edit the value or change the state of the radio.

A Radio Button with the prop `readOnly="true"` will suppress all functionality along with displaying read-only styling.

</LivePreview>

<LivePreview componentName="radio-button" exampleName="Warning">

### Validation states: warning

You can indicate a warning state stylistically by setting the `validationStatus` prop to "warning".

- When used in a group, use `validationStatus` on the group rather than the individual controls. Any status provided to the group will take precedence over the statuses applied directly to the nested controls.
- Avoid using `validationStatus` when a Radio Button is disabled. If provided, disabled functionality and styling will take precedence over any validation status styling.

</LivePreview>

### Long text support

Radio Button supports long values. If a text value wraps to multiple lines, align the text to the top of the Radio Button.

</LivePreview>

### Form Field compatibility

Radio Button group can be wrapped in a Form Field when it’s displayed within a Form context. This provides functionality built into Form Field for increased accessibility.

For more information, refer to the [Form Field](https://www.saltdesignsystem.com/salt/patterns/forms) guidance.
