## Best practices

### Triggering mega menu

- Any button that triggers a Mega menu container.
- It should be wrapped in a `<nav>` element to establish a navigation landmark.
- Structure the group of buttons as a list, with each button in its own `<li>`. Use a `<ul>`, since the triggers have no inherent order. Reserve `<ol>` for cases where the sequence is meaningful and must be announced to assistive technology.
- `MegaMenuTrigger` automatically applies **aria-expanded** to reflect the expanded or collapsed state, and **aria-controls** to reference the associated mega menu panel.

### Mega menu panel

- `MegaMenuPanel` renders a `<div>` with `role="region"`. Always provide an `aria-label` on `MegaMenuPanel` that describes the menu content, for example `aria-label="Solutions menu"`. This ensures assistive technology users can distinguish the panel from other landmarks on the page.

### Custom region

- A custom region can include a locally customized component; however, it must comply with ADA standards. For example, opening another modal through an action in a custom region may violate ADA compliance and should be avoided.

### Accessibility semantics

- The Mega menu container should use **role="region"**.
- Mega menu groups should use **Link lists**, structured as `<ul>` elements containing `<li>` items.
- Each `<ul>` should have an accessible name that corresponds to the group label.

## Keyboard interactions

<KeyboardControls>
<KeyboardControl keyOrCombos="Enter / Space / Down arrow">

- When the trigger has focus, pressing Enter, Space, or the Down arrow opens the mega menu.
- The Down arrow additionally moves focus to the first mega menu item.

</KeyboardControl>

<KeyboardControl keyOrCombos="Escape">

- If focus is in the mega menu, Escape closes the mega menu and returns focus to the triggering element.

</KeyboardControl>

<KeyboardControl keyOrCombos="Tab">

- Focus starts on the first mega menu item and moves sequentially through each item in a group, then continues to the first item of the next group once the last item is reached.
- Tab includes interactive elements within custom regions in the focus sequence, based on their position in the layout. The custom region itself is not focusable.
- After the last focusable element, Tab moves focus out of the mega menu and collapses it.

</KeyboardControl>

<KeyboardControl keyOrCombos="Shift + Tab">

- If focus is in the mega menu, Shift+Tab sets focus to the previous focusable component in the tab order.
- If the first element in the mega menu has focus, Shift+Tab moves focus to the previous focusable element outside of the mega menu.

</KeyboardControl>

<KeyboardControl keyOrCombos="Up arrow">

- Moves focus to the previous item within a column.
- On the first item of a column, moves focus to the trigger and keeps the menu open.
- In the supporting actions row, returns focus to the last column.

</KeyboardControl>

<KeyboardControl keyOrCombos="Down arrow">

- Moves focus to the next item within a column.
- On the last item of a column, moves focus to the first item of the next column.
- On the last item of the last column, moves into the supporting actions row if one is present; otherwise it moves focus to the next trigger and closes the menu, or does nothing when there is no next trigger.
- In the supporting actions row, the last action moves focus to the next trigger and closes the menu, or does nothing when there is no next trigger.

</KeyboardControl>

<KeyboardControl keyOrCombos="Left arrow">

- Moves focus to the first item of the previous column.
- On the first column, moves focus to the trigger and keeps the menu open.
- In the supporting actions row, moves focus to the previous action.

</KeyboardControl>

<KeyboardControl keyOrCombos="Right arrow">

- Moves focus to the first item of the next column.
- On the last column, moves focus to the trigger and keeps the menu open — except on the last item, where it moves focus to the next trigger and closes the menu when one exists.
- In the supporting actions row, moves focus to the next action; on the last action it behaves like the last item of the last column.

</KeyboardControl>

<KeyboardControl keyOrCombos="Home / End">

- Home moves focus to the first item in the current group, and End to the last.
- In the supporting actions row, they move to the first and last action.

</KeyboardControl>
</KeyboardControls>
