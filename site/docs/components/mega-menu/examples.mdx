## Default

The mega menu displays grouped navigation links in a multi-column layout sized to hug its contents. It’s typically opened via a trigger button and can be closed by clicking outside the menu, pressing Escape key, or clicking the trigger again. Each mega menu item links to a specific page. If the content exceeds the available space, the menu wraps in width and scrolls vertically.

When the content becomes wider than the available width, the container expands until it reaches the inset defined by the left and right Layout-page-margin, and the mega menu groups wrap to fit.

## Full width

The mega menu container is full width within the inset, expanding to the left and right Layout-page-margin by default. The content then fills the container.

## Edge to edge

The mega menu container expands to the full viewport width (edge-to-edge). In this example, the content is center-aligned and constrained to a maximum width, creating balanced left/right margins on larger displays.

## In small viewport

When the mega menu is triggered in a small viewport, or the screen size changes to a small viewport, the expanded mega menu arranges its groups, Contents, and links in a stacked layout, preserving their original order. This ensures the menu remains accessible and easy to navigate on smaller screens.

### Best practices

- Although the mega menu stacks into a single column on small screens, it’s more suitable for desktop. In small viewports it can become hard to scan, scroll-heavy, and more error-prone on touch.

- If content is large or frequently used on mobile, use an alternate mobile nav (e.g., drawer/accordion).

## With adornment

You can display adornments such as tags, badges, or icons. This allows you to indicate statuses, show counts, or signal external navigation.

### Best practices

- Only one component—such as a tag or badge should be displayed per item.
- When using badge components, it’s recommended not to mix default badges and dot badges within the same mega menu container.

A content can be positioned on the left or right, allowing for flexible layouts like banners or images alongside mega menu items. In small viewports, it reflows into the stacked mega menu in its original order, appearing at the top or bottom of the stack.
