## Default

The mega menu displays grouped menu items in a multi-column layout. The menu container is typically triggered by a button and can be closed by clicking outside the menu, pressing the Escape key, or clicking the trigger component again. Each menu item acts as a link that navigates to a specific page. If the content exceeds the available space, the menu becomes scrollable.

## With Adornment

You can display adornments such as tags, badges, or icons. This allows you to indicate statuses, show counts, or signal external navigation.

### Best practices

- Only one component—such as a tag or badge should be displayed per item.
- When using badge components, it’s recommended not to mix default badges and dot badges within the same mega menu container.

## In small viewport

When the mega menu is triggered in a small viewport, or the screen size changes to a small viewport, the expanded mega menu arranges its groups, custom regions, and links in a stacked layout, preserving their original order. This ensures the menu remains accessible and easy to navigate on smaller screens.
