Density relates to the amount of content that can fit on a screen based on the size and spacing of components. When building a product with Salt, select the density that best suits your use case. All components, spacing, and typographic elements will align and remain proportionate regardless of the density you choose.

Read about implementing density in code on the [SaltProvider](/salt/components/salt-provider/examples) documentation.

## Desktop densities

Desktop densities are tailored for different types of applications, from complex data platforms to content-focused interfaces. Select the density that best fits your app or websites complexity and user needs.

| Densities | Use case                                                                                                                               |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Low       | Best for data-light or content-focused interfaces, such as marketing and communications, where a spacious layout enhances readability. |
| Medium    | Suited for standard applications and websites, providing a balanced default density for most use cases.                                |
| High      | Ideal for data-heavy or complex applications, such as trading platforms, where maximum information needs to be displayed efficiently.  |

High density is suitable for specialized use cases. It’s ideal for data-dense applications where screen real estate is maximized, or for applications frequently used by experienced users.

<Callout
  status="warning"
  title="High density is not recommended as the default density"
>
High density doesn’t meet [WCAG 2.2](https://www.w3.org/WAI/WCAG22/Understanding/) level AA requirements, including target size. When offering a choice of densities, users must be informed about the accessibility implications prior to applying high density. All information and functionality must remain available regardless of the density selected—switching density must not change the experience beyond presentation (e.g., spacing, text size).

Read about [Guiding conscious decisions](/salt/patterns/experience-customization#guiding-conscious-decisions) on the Experience customization pattern.

</Callout>

## Touch density

Touch density’s main purpose is to address accessibility concerns on smaller viewports that deal with higher resolutions. When a screen has a higher resolution, touch targets are adjusted to remain large enough for easy interaction.

<Callout
  status="warning"
  title="Touch density deprecation"
>
Touch density will be removed in the next breaking change.

It was originally introduced to support basic touch accessibility, but is not optimized for smaller screens—as density increases, components and spacing become larger, leading to inefficient layouts and usability issues on mobile devices.

What you should do:

- For mobile or tablet experiences, migrate to mobile density.
- For desktop or web experiences, migrate to low density.

Refer to the [Migrating from touch to mobile density](https://go/saltds/content/news/20260204-migrating-from-touch-to-mobile-density) (Internal users only) for step-by-step instructions.

</Callout>

## Choosing density

Select a density early in the design process to meet your application's requirements. Salt’s scaling system ensures components fit together seamlessly when using a consistent density. Mixing densities can cause alignment issues and reduce overall design quality.

## Switching density

In some cases, you may want to switch density to optimize the experience for different devices or screen sizes. There are two main approaches:

1. By screen size (breakpoints): change density as the viewport crosses defined breakpoints. For more information on breakpoints, see the [breakpoints](/salt/foundations/responsiveness/breakpoints) documentation.

2. By device type: change density depending on whether the user is on a mobile, tablet, or desktop device. See an example of this in the [SaltProvider](/salt/components/salt-provider/examples) documentation.
