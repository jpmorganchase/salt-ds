## What to expect

<Impact
  impacts={[
    {
      title: "Small components, flexible patterns",
      description:
        "We know we can't build components for every use case across every line of business, so we focus on building small, quality components and then demonstrate how to put them together as pattern examples which address specific business requirements.",
    },
    {
      title: "Lightweight, dependency-light",
      description:
        "We aim to keep Salt as lightweight as possible, with minimal dependencies. We've built our components from the ground up to be customizable and easy to keep up to date.",
    },
    {
      title: "Composable APIs",
      description:
        "We have looked at the friction points created by legacy APIs to make them more consistent an composable. We can't build component features for all use cases but with our APIs you can compose other elements in Salt components to address your specific user interaction.",
    },
    {
      title: "A systematic spacing grid",
      description:
        "We build all of our components to support our four systems of density by using a 4px scaling system that allows you to use components that 'fit' together cohesively at a size which is optimal for the amount of data you have on the screen.",
    },
    {
      title: "WCAG 2.1 accessibility",
      description:
        "We follow Web Content Accessibility Guidelines (WCAG) 2.1 to level AA standards. Salt components are developed with accessibility as a core requirement, not an afterthought. We believe in digital inclusion an are committed to supporting the diverse needs of all users.",
    },
    {
      title: "Token-driven styling",
      description:
