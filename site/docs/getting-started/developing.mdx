Learn how to use Salt’s UI components in your React web applications.

## What you’ll need

To start developing with Salt, you should:

- Be familiar with [React](https://react.dev/) and web development fundamentals (such as HTML, CSS and JavaScript).
- Have [Node.js](https://nodejs.org/) (version 22 or later) installed on your system.
- Create a React (version 18 or later) web application project. For new projects, see the React docs for [Creating a React App](https://react.dev/learn/creating-a-react-app).

## The step-by-step process

### 1. Install the Salt packages

The three main packages you’ll need are:

- `@salt-ds/core` contains production-ready UI components.
- `@salt-ds/theme` contains the CSS for Salt’s themes.
- `@salt-ds/icons` contains SVG-based icons.

Install all three in one go:

```bash
npm install @salt-ds/core @salt-ds/theme @salt-ds/icons
```

If you prefer not to use npm, you can install the packages using [yarn](https://yarnpkg.com/) or other package managers. Salt ships additional packages for specific use cases—such as [`@salt-ds/lab`](#5-optional-play-with-lab-components) for experimental components, plus charting, data grid and date utilities—which you can add as you need them.

### 2. Add required web fonts

Salt requires the Open Sans and PT Mono web fonts. You can integrate them into your application using one of the following methods.

If you’re planning to use the JPM Brand theme (recommended in step 3), you’ll also need the **Amplitude** web font—see [Setting up the JPM Brand theme](../themes#setting-up-the-jpm-brand-theme) in Themes for where to obtain the font files and the `@font-face` declarations.

#### Use Fontsource (recommended)

With Fontsource, you can include the web font files into your build and self-host them. This is the recommended approach for enterprise applications because it avoids third-party requests at runtime.

First, install the relevant npm packages:

```bash
npm install @fontsource/open-sans @fontsource/pt-mono
```

Next, import the CSS for the weights and styles that Salt requires. In a typical React project setup, the best place to put these imports is in your `src/index.tsx` file:

```tsx
import "@fontsource/open-sans/300.css";
import "@fontsource/open-sans/300-italic.css";
import "@fontsource/open-sans/400.css";
import "@fontsource/open-sans/400-italic.css";
import "@fontsource/open-sans/500.css";
import "@fontsource/open-sans/500-italic.css";
import "@fontsource/open-sans/600.css";
import "@fontsource/open-sans/600-italic.css";
import "@fontsource/open-sans/700.css";
import "@fontsource/open-sans/700-italic.css";
import "@fontsource/open-sans/800.css";
import "@fontsource/open-sans/800-italic.css";
import "@fontsource/pt-mono";
```

</Disclosure>

Learn more about [Fontsource](https://fontsource.org/).

#### Use Google Fonts

To load the fonts directly from Google Fonts, add the following into your application’s `index.html` file:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Open+Sans:ital,wght@0,300;0,400;0,500;0,600;0,700;0,800;1,300;1,400;1,500;1,600;1,700;1,800&family=PT+Mono&display=swap"
  rel="stylesheet"
/>
```

Refer to the [Google Fonts page](https://fonts.google.com/share?selection.family=Open+Sans:ital,wght@0,300;0,400;0,500;0,600;0,700;0,800;1,300;1,400;1,500;1,600;1,700;1,800|PT+Mono) for more information.

#### Use another method

If you prefer not to use Fontsource or Google Fonts, you can use any other method to incorporate the web fonts. Make sure the CSS `@font-face` declarations associate the correct `font-family`, `font-weight` and `font-style` with the corresponding web fonts.

### 3. Choose your theme and integrate Salt

The `@salt-ds/theme` package ships two themes:

- **JPM Brand**—_recommended for all new projects._ Salt’s long-term visual style, aligned with the JPMorgan brand.
- **Legacy (UITK)**—for applications migrating from UITK that need to preserve the existing visual style.

Additional themes, such as Chase, are distributed as separate packages. See [Themes](../themes) for a full comparison.

Whichever theme you choose, you’ll import the theme CSS and wrap your application in `<SaltProvider>` or `<SaltProviderNext>`.

#### Using the JPM Brand theme (recommended)

Make sure you’ve added the **Amplitude** web font (see [step 2](#2-add-required-web-fonts)). In a typical React project, set up the provider in your `src/index.tsx` as follows:

```tsx
// Import <SaltProviderNext>
import { SaltProviderNext } from "@salt-ds/core";
import ReactDOM from "react-dom/client";
import App from "./App";

// Import the base theme CSS plus the JPM Brand layer
import "@salt-ds/theme/index.css";
import "@salt-ds/theme/css/theme-next.css";

// Wrap your application in a <SaltProviderNext>
const container = document.getElementById("root");
if (!container) throw new Error("Failed to find the root element");
const root = ReactDOM.createRoot(container);
root.render(
  <SaltProviderNext
    accent="teal"
    corner="rounded"
    headingFont="Amplitude"
    actionFont="Amplitude"
  >
    <App />
  </SaltProviderNext>,
);
```

<Disclosure
  id="using-the-legacy-uitk-theme"
  summary="Using the Legacy theme"
>

Use this provider if you’re migrating from UITK and need to preserve the existing visual style.

```tsx
// Import <SaltProvider>
import { SaltProvider } from "@salt-ds/core";
import ReactDOM from "react-dom/client";
import App from "./App";

// Import theme CSS
import "@salt-ds/theme/index.css";

// Wrap your application in a <SaltProvider>
const container = document.getElementById("root");
if (!container) throw new Error("Failed to find the root element");
const root = ReactDOM.createRoot(container);
root.render(
  <SaltProvider>
    <App />
  </SaltProvider>,
);
```

</Disclosure>

#### Set mode and density

Both `<SaltProvider>` and `<SaltProviderNext>` accept `mode` and `density` props that control the overall look and feel of your application:

- `mode`—`"light"` (default) or `"dark"`. See [Modes](../foundations/modes).
- `density`—`"high"`, `"medium"` (default), `"low"`, `"touch"` or `"mobile"`. See [Density](../foundations/density).

Add these props to your provider, alongside any theme props from earlier:

```tsx
<SaltProviderNext mode="dark" density="low">
  <App />
</SaltProviderNext>
```

Refer to the [SaltProvider](../components/salt-provider) docs for the full API.

### 4. Import components

Now that you’re ready to use our UI components, you can import them from the `@salt-ds/core` package and use them in your JSX code.

For example:

```tsx
import { Button } from "@salt-ds/core";
import { AddIcon } from "@salt-ds/icons";

function App() {
  return (
    <Button>
      <AddIcon aria-hidden /> Click me
    </Button>
  );
}

export default App;
```

Head over to our available [components](../components) to view their respective APIs.

### 5. (Optional) Play with lab components

Our `@salt-ds/lab` package is where we introduce new components that are in the initial stages of development. Once they’re stable, we move them over to the core package.

Because it’s an unstable environment that may cause major breaking changes, we don’t recommend using lab components in production applications. However, early adopters are welcome to use and experiment with them.

To try them out, install the package:

```bash
npm install @salt-ds/lab
```
