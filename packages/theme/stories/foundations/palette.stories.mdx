# Palette

The palette sits in between the foundations and characteristics. The palette is a set of refined color tokens using values from the color foundation, and a few tokens with semantic meaning given to the respective opacity foundation. This set is split into subgroups guided by semantics:

| Semantic | Appropriate usage                                               | Example token                               | UITK Theme value<br /> (light mode) |
| -------- | --------------------------------------------------------------- | ------------------------------------------- | ----------------------------------- |
| Accent   | Characteristics which use the main brand color                  | `--uitk-palette-accent-background`          | `--uitk-color-blue-500`             |
| Interact | Characteristics relating to actions such as editing or clicking | `--uitk-palette-interact-border`            | `--uitk-color-gray-90`              |
| Error    | Characteristics symbolising an error state                      | `--uitk-palette-error-border`               | `--uitk-color-red-500`              |
| Info     | Characteristics symbolising an info state                       | `--uitk-palette-info-foreground`            | `--uitk-color-blue-500`             |
| Success  | Characteristics symbolising a success state                     | `--uitk-palette-success-border`             | `--uitk-color-green-500`            |
| Warning  | Characteristics symbolising a warning state                     | `--uitk-palette-warning-foreground`         | `--uitk-color-orange-500`           |
| Measure  | Characteristics showing some form of progress                   | `--uitk-palette-measured-fill`              | `--uitk-color-blue-500`             |
| Navigate | Characteristics allowing for navigation                         | `--uitk-palette-navigate-foreground`        | `--uitk-color-gray-400`             |
| Negative | Characteristics conveying a negative value or result            | `--uitk-palette-negative-foreground`        | `--uitk-color-red-700`              |
| Neutral  | Characteristics with no specialised semantic meaning            | `--uitk-palette-neutral-primary-foreground` | `--uitk-color-gray-900`             |
| Positive | Characteristics conveying a positive value or result            | `--uitk-palette-positive-foreground`        | `--uitk-color-green-700`            |
| Rate     | Characteristics allowing for something to be rated              | `--uitk-palette-rate-border-undo`           | `--uitk-color-orange-600`           |

# Usage

The palette provides meaning to the colors used throughout the characteristics. For a component with a background color set to `--uitk-actionable-cta-background`, rather than this token directly pointing to the foundation token `--uitk-color-blue-500`, this value instead resolves to the palette token `--uitk-interact-cta-background`. Similarly, selectable backgrounds resolve to the same palette token. This ensures that anything that is interactable throughout your theme is consistent. The palette sits directly in the middle.

Component CSS attribute &#8594; Characteristic token &#8594; **_Palette token_** &#8594; Foundation token &#8594; Raw RGBA Value

# Token structure

Palette tokens are structured as follows: `--uitk-palette-<sentiment>-<variant>-<usage>-<interaction>-<emphasis>`

- Always begin with `--uitk-palette-`
- Follow this by the **_semantic meaning_**, e.g. `---uitk-palette-accent-`, `---uitk-palette-interact-`, `--uitk-palette-success-`, `--uitk-palette-neutral-`
- Append the **_variant_**, if applicable: e.g. `--uitk-palette-interact-cta-`
- Follow this by **_usage_** : e.g. `--uitk-palette-accent-background`, `--uitk-palette-interact-cta-background-`, `--uitk-palette-success-foreground`, `--uitk-palette-neutral-border-`
- Next add the **_interaction_**, if applicable : e.g. `--uitk-palette-interact-cta-background-hover`
- Finally, add **_emphasis_** if applicable, e.g.: `--uitk-palette-neutral-border-medium`

# Emphasis

Some tokens below have `-low`, `-medium`, or `-high` suffixes. These are values which should change with emphasis. For example,

The characteristic `--uitk-overlayable-background-medium` resolves to `--uitk-palette-neutral-scrim-medium` whilst the characteristic `--uitk-overlayable-background-low` resolves to `--uitk-palette-neutral-scrim-low`.

The characteristic with the respective emphasis should be used within a `uitkEmphasisLow`, `uitkEmphasisMedium`, or `uitkEmphasisHigh` class. For example, if the default input in your app should have a medium emphasis background, but can be styled with low emphasis, your CSS will probably look something like this:

```css
.MyInputComponent {
  background: var(--uitk-editable-background-medium);
}

.MyInputComponent:hover {
  background: var(--uitk-editable-background-hover-medium);
}

.uitkEmphasisLow.MyInputComponent {
  background: var(--uitk-editable-background-low);
}

.uitkEmphasisLow.MyInputComponent:hover {
  background: var(--uitk-editable-background-hover-medium);
}
```
