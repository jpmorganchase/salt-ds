# Palette

There is a special layer which sits in between the foundations and characteristics: the palette. The palette is a set of refined color tokens using values from the color foundation, and a few tokens with semantic meaning given to the respective opacity foundation. This set is split into subgroups guided by semantics:

| Semantic | Appropriate usage                                               | Example token                                            | UITK Theme value<br /> (light mode) |
| -------- | --------------------------------------------------------------- | -------------------------------------------------------- | ----------------------------------- |
| Accent   | characteristics which use the main brand color                  | `--uitk-palette-accent-background`                       | `--uitk-color-blue-500`             |
| Interact | characteristics relating to actions such as editing or clicking | `--uitk-palette-interact-border`                         | `--uitk-color-grey-90`              |
| Error    | characteristics symbolising an error state                      | `--uitk-palette-error-border`                            | `--uitk-color-red-500`              |
| Info     | characteristics symbolising an info state                       | `--uitk-palette-info-icon`                               | `--uitk-color-blue-500`             |
| Success  | characteristics symbolising a success state                     | `--uitk-palette-success-border`                          | `--uitk-color-green-500`            |
| Warning  | characteristics symbolising a warning state                     | `--uitk-palette-warning-icon`                            | `--uitk-color-orange-500`           |
| Measure  | characteristics showing some form of progress                   | `--uitk-palette-measure-background-gradient-start-color` | `--uitk-color-teal-500`             |
| Navigate | characteristics allowing for navigation                         | `--uitk-palette-navigate-foreground`                     | `--uitk-color-grey-400`             |
| Negative | characteristics conveying a negative value or result            | `--uitk-palette-negative-foreground`                     | `--uitk-color-red-700`              |
| Neutral  | characteristics with no specialised semantic meaning            | `--uitk-palette-neutral-primary-foreground`              | `--uitk-color-grey-900`             |
| Positive | characteristics conveying a positive value or result            | `--uitk-palette-positive-foreground`                     | `--uitk-color-green-700`            |
| Rate     | characteristics allowing for something to be rated              | `--uitk-palette-rate-border-undo`                        | `--uitk-color-orange-600`           |

# Usage

The palette is intended to provide meaning to the colors used throughout the characteristics. Whilst a component only has to care that their background color is `--uitk-actionable-cta-background`, rather than this token directly pointing to the foundation token `--uitk-color-blue-500`, this value instead resolves to the palette token `--uitk-interact-cta-background`. Similarly, selectable backgrounds result to the same palette token. This ensures anything that is interactable throughout your theme is consistent. The palette sits directly in the middle.

Component CSS attribute &#8594; Characteristic token &#8594; **_Palette token_** &#8594; Foundation token &#8594; Raw RGBA Value

# Token structure

Palette tokens are structured as follows: `--uitk-palette-<sentiment>-<variant>-<usage>-<interaction>-<emphasis>`

1. Always begin with `--uitk-palette-`
2. Follow this by the **_semantic meaning_**, e.g. `---uitk-palette-accent-`, `---uitk-palette-interact-`, `--uitk-palette-success-`, `--uitk-palette-neutral-`
3. Append the **_variant_**, if applicable: e.g. `--uitk-palette-interact-cta-`
4. Follow this by **_usage_** : e.g. `--uitk-palette-accent-background`, `--uitk-palette-interact-cta-background-`, `--uitk-palette-success-foreground`, `--uitk-palette-neutral-border-`
5. Next add the **_interaction_**, if applicable : e.g. `--uitk-palette-interact-cta-background-hover`
6. Finally, add **_emphasis_** if applicable, e.g.: `--uitk-palette-neutral-border-medium`

# Emphasis

Some tokens below have `-low`, `-medium`, or `-high` suffixes. This means that they are values which should change with emphasis. For example,

The characteristic `--uitk-overlayable-background-medium` resolves to `--uitk-palette-neutral-scrim-medium` whilst the characteristic `--uitk-overlayable-background-low` resolves to `--uitk-palette-neutral-scrim-low`.

The characteristic with the respective emphasis should be used within a `uitkEmphasisLow`, `uitkEmphasisMedium`, or `uitkEmphasisHigh` class. For example, if the default input in your app should have a medium emphasis background, but can be styled with low emphasis, your CSS will probably look something like this:

```css
.MyInputComponent {
  background: var(--uitk-editable-background-medium);
}

.MyInputComponent:hover {
  background: var(--uitk-editable-background-hover-medium);
}

.uitkEmphasisLow.MyInputComponent {
  background: var(--uitk-editable-background-low);
}

.uitkEmphasisLow.MyInputComponent:hover {
  background: var(--uitk-editable-background-hover-medium);
}
```
