# About Foundations

The foundations lie at the heart of the Salt design system, and define the core visual elements needed to create a consistent experience.

The design tokens within each foundation make up the building blocks of the Salt theme.

## Token name convention

Foundation tokens follow the format `--salt-<foundation name>-<variant>-<scale>-<density>`

1. Always begin with `--salt-`
2. Followed by the **foundation name**, e.g. `--salt-color-`, `--salt-shadow-`, `--salt-size-`, `--salt-typography-`
3. Append the **intent**, if applicable: e.g. `--salt-color-blue-`, `--salt-shadow-elevation-`, `--salt-size-stackable`, `--salt-typography-fontWeight-`
4. Append the **scale**, if applicable: e.g. `--salt-color-blue-700`, `--salt-shadow-elevation-3`, `--salt-typography-fontWeight-light`

## Foundation tokens

Foundation tokens are divided into the following groups:

| Foundation            | Description                                                         | Example token                          | Example value                     |
| --------------------- | ------------------------------------------------------------------- | -------------------------------------- | --------------------------------- |
| Color                 | The core set of colors used in the Salt theme                       | `--salt-color-gray-800`                | `rgb(36, 37, 38)`                 |
| Duration              | Duration between a users actions and system feedback                | `--salt-duration-cutoff`               | `10000`                           |
| Icon                  | The core set of icons available in the Salt theme                   | `--salt-icon-size-base`                | `12px`                            |
| Opacity               | Opacity percentages for different attributes                        | `--salt-opacity-background`            | `0.7`                             |
| Shadow                | Shadows used to convey elevation or hierarchy of components         | `--salt-shadow-black-4`                | `0 6px 10px 0 rgba(0, 0, 0, 0.2)` |
| Size                  | Size in px, used for padding, height, border width, etc.            | `--salt-size-basis-unit`               | `4px`                             |
| Spacing               | Spacing in px used for padding, margins, etc., based on spacing-100 | `--salt-spacing-100`                   | `4px`                             |
| Typography            | The fonts and font weights provided by the theme                    | `--salt-typography-fontWeight-regular` | `400`                             |
| Z-Index               | Numbering indicating appropriate z-index of components              | `--salt-zIndex-flyover`                | `1500`                            |
| **Deprecated:** Delay | Replaced with Duration                                              | Replaced with Duration                 | Replaced with Duration            |

## Using Foundations

Wrap the foundation you want to use inside a `var()` function. Most commonly, foundations are used when referenced by characteristics, for example:

```css
.salt-theme {
  --salt-actionable-primary-fontWeight: var(--salt-typography-fontWeight-bold);
}
```

where the **actionable characteristic** `--salt-actionable-primary-fontWeight` references the variable `--salt-typography-fontWeight-bold` from the **typography foundation**.

However, foundations can also be used directly if there is no appropriate characteristic provided for your use case. For example, you could color your headings green:

```css
.h1 {
  color: var(--salt-color-green-100);
}
```

or, even inline!

```html
<h1 style={{ color: "var(--salt-color-green-100)" }}>Example</h1>
```
