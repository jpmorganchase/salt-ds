# Foundations

The foundations lie at the heart of the Salt design system, and define the core visual elements needed to create a consistent experience. 

The design tokens within each foundation make up the building blocks of the Salt theme.

## Naming tokens

Foundation tokens follow the format `--uitk-<foundation name>-<variant>-<scale>-<density>`

1. Always begin with `--uitk-`
2. Followed by the **foundation name**, e.g. `---uitk-color-`, `---uitk-shadow-`, `--uitk-size-`, `--uitk-typography-`
3. Append the **intent**, if applicable: e.g. `--uitk-color-blue-`, `--uitk-shadow-elevation-`, `--uitk-size-stackable`, `--uitk-typography-fontWeight-`
4. Append the **scale**, if applicable: e.g. `--uitk-color-blue-700`, `--uitk-shadow-elevation-3`, `--uitk-typography-fontWeight-light`

## Foundation tokens

Foundation tokens are divided into the following groups:

| Foundation  | Description   | Example token   | Example value   |
|---          |---            |---              |---              |
| Color       | The core set of colors used in the Salt theme               | `--uitk-color-gray-800`                 | `rgb(36, 37, 38)` |
| Delay       | Duration between a users actions and system feedback        | `--uitk-delay-cutoff`                   | `10000`           |
| Icon        | The core set of icons available in the Salt theme           | `--uitk-icon-size-base`                 | `12px`            |
| Opacity     | Opacity percentages for different attributes                | `--uitk-opacity-background`             | `0.7`             |
| Shadow      | Shadows used to convey elevation or hierarchy of components | `--uitk-shadow-black-4`              | `0 6px 10px 0 rgba(0, 0, 0, 0.2)` |
| Size        | Size in px, used for padding, height, border width, etc.    | `--uitk-size-basis-unit`                | `4px`             |
| Typography  | The fonts and font weights provided by the theme           | `--uitk-typography-fontWeight-regular`  | `400`             |
| Z-Index     | Numbering indicating appropriate z-index of components      | `--uitk-zIndex-flyover`                 | `1500`            |


## Using Foundations

Wrap the foundation you want to use inside a `var()` function. Most commonly, foundations are used when referenced by characteristics, for example:

```css
.uitk-theme {
  --uitk-actionable-primary-fontWeight: var(--uitk-typography-fontWeight-bold);
}
```

where the **actionable characteristic** `--uitk-actionable-primary-fontWeight` references the variable `--uitk-typography-fontWeight-bold` from the **typography foundation**.

However, foundations can also be used directly if there is no appropriate characteristic provided for your use case. For example, you could color your headings green:

```css
.h1 {
  color: var(--uitk-color-green-100);
}
```

or, even inline!

```html
<h1 style={{ color: "var(--uitk-color-green-100)" }}>Example</h1>
```

### Using the Size Foundation

Size foundations are commonly used within a calculation for settings such as width, height, paddings, margins. For example:

```css
.uitkLogo {
  height: calc(1.5 * var(--uitk-size-unit)));
}
```
