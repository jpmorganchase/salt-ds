# Icon

Icon is an important visual representation of an application, a capability, or some other concept or specific entity with meaning on a interface. Icons help users to recognize and understand a concept or an action regardless of linguistic or cultural boundaries.

If an icon is interactive, use [Button](?path=/docs/documentation-core-button--button-variants) (`CTA`, `Primary` or `Secondary` variant).

<Canvas>
  <Story id="icons-icon--toolkit-icon" />
</Canvas>

## Use Icon when...

- You need a visual representation of a concept.
- An icon doesn't need to be interactive.

## Features

We have a default icon set that starts from 12px for high and medium density (MD), then 14px for low density (LD), and 16px for touch density (TD). (Since our base size of high density (HD)/default is 10px but we don’t allow icons size smaller than 12px, hence why here the HD/default will start from 12px, and its 2X is 20px, 3X is 30px etc.)
You have more size choices by adding multiplies to 2x, 3x, 4x etc.

  <Story id="icons-icon--toolkit-icon-multiple-sizes" />
</Canvas>

We provide two types of icons: default and solid. Some special icons may be excluded from it like `plus` and `loader`.

## Accessibility

To ensure that icon-only buttons are ADA compliant, when icon is inside a button you must display the button’s label in a supporting Tooltip that is revealed on mouse hover or keyboard focus.
The icon name/description should be read out automatically by a screen-reader with the Aria label.
