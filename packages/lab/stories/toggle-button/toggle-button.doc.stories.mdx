# Toggle Button

<Canvas>
  <Story id="lab-toggle-button--text-and-icon" />
</Canvas>

This basic component allows users to enable or disable a single command.

### Text and Icon

Use icons and text in Toggle Buttons if it's important to visually reinforce the theme of the options or if providing additional context may help decision-making (for examples, selecting a document type or a text style).

- Pass `aria-hidden=”true”` to the icon so its label is not announced by the screen reader.

<Canvas>
  <Story id="lab-toggle-button--text-and-icon" />
</Canvas>

### Icon only

The Toggle Button Group can be used with icons only.

- An accessible label should be provided for each button.
- A `Tooltip` should be used to provide additional context for each `ToggleButton`.

## How to use

### When not to use Toggle Button

- If you're presenting a binary choice between two alternatives, consider a Switch. This choice is especially relevant if the execution is instantaneous and isn't required to be submitted first.
- If the option is likely to change depending on context (for example, user filter selection), use a selectable Pill instead.

## Accessibility

The Toggle Button is accessible by screen readers and provides full WAI-ARIA Checkbox support.

### Keyboard interactions

| Key                                  | Function                                                                                                                                            |
| ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| <kbd>Enter</kbd> or <kbd>Space</kbd> | When focus is on the Toggle Button and the button is inactive, activates the button. If focus is on an active button, it makes the button inactive. |
