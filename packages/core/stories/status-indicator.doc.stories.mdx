# Status Indicator

A status indicator can be used on its own or within another component (e.g. `Tooltip`) to help convey a message.

<Canvas>
  <Story id="core-statusindicator--default-status-indicator" />
</Canvas>

## How you can use status indicator

Use this component by passing in your chosen `status` from the available options.

You can also customize the indicator's `size`.

## Configuring status indicator

### API

```
import { StatusIndicator } from "@jpmorganchase/uitk-core";
```

### Props

You can set the indicator status and other configurtions via its props.

## Features

### Status indicators

The status indicator offers four severity levels that set a distinctive indicator and color.

<Canvas>
  <Story id="core-statusindicator--all-status-indicators" />
</Canvas>

### Use when...

An `error` indicator should be used when there is critical information preventing an action.

A `warning` or `info` (depending on the severity) indicator should be used when there is important, but non blocking, information to be conveyed to the user.

Use a `success` indicator to confirm that an action has been completed.

## Accessibility

Decorative status indicators (status indicators that repeat the information already expressed by the accompanied text) should be hidden from the screen reader to avoid repetition.
If you are adding a status indicator for decorative purposes, pass `aria-hidden=”true”` to hide its label from the screen reader.
