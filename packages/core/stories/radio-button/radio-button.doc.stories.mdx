# Radio Button and Radio Button Group

A group of Radio Button components represents a set of mutually exclusive, but related values. The user can select a single value at a time, and if they select an unselected value it will automatically deselect any previously-selected option. Radio Buttons should therefore be used to display a choice between two or more values, as it’s impossible to deselect a single Radio Button once it’s been selected.

Error state can be indicated stylistically by setting the `validationStatus` prop to "error".

Error state can be indicated stylistically by setting the `error` prop to true.

- When used in a group, use `validationStatus` on the group rather than the individual controls. Any status provided to the group will take precedence over the statuses applied directly to the nested controls.
- Avoid using `validationStatus` when a Radio Button is `disabled`. If provided, `disabled` functionality and styling will take precedence over any validation status styling.

## Radio Button Group

Radio groups allow users to select one or more values from a given set of choices. They can be aligned horizontally or vertically.

**Best practices**

- Always use a label for a Radio Group.
- If there are more than five choices, we suggest you use a multiselect instead.

#### Wrapping

By default, horizontal radio groups will wrap onto the next line if the group doesn't fit within the container. You can force the group to remain inline and wrap the label text instead by setting the `wrap` prop to `false`.

### Controlled

Controlled Radio buttons require events to trigger the state change. The application must track state and update Radio props manually. Useful for when logic or validation needs to be applied or a request should happen when updating a form.
