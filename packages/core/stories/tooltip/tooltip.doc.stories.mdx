# Tooltip

A Tooltip provides guidance to the user when something requires more explanation or context, or to provide more information about an error or warning condition. A Tooltip will typically be displayed after the user’s mouse hovers over the target element for a set amount of time.

## Best practise

- Content within tooltips should be supplementary information that provides succinct and descriptive clarity to the user.
- Do not place information necessary for the user to complete a task inside a tooltip. Any important help text should remain visible on screen at all times, not just when a tooltip is displayed.

## Use when...

You want to display extra information or a description of an on-screen element.

## Don't use when...

- **The Tooltip contains interactive elements like link, input or button.** Consider making this content visible on the page or use an Overlay or a Dialog instead.
- **There is a lot of content inside the Tooltip.** Tooltip content should be short and self-explanatory. If the text needs to be more descriptive, consider adding it to the page so it's always visible to the user.

## Features

### Status

Use the `status` prop to choose between `error`, `warning`, `success` and `info`. This defines which icon will be displayed in the Tooltip.

<Canvas>
  <Story id="core-tooltip--status" />
</Canvas>

### Without arrow

If you'd like to display the Tooltip without the arrow, use the `hideArrow` prop.

<Canvas>
  <Story id="core-tooltip--without-arrow" />
</Canvas>

### Without icon

To display the Tooltip with text only, and no status icon, use the `hideIcon` prop.

<Canvas>
  <Story id="core-tooltip--without-icon" />
</Canvas>

### Content

Use the `content` prop to pass either a simple string or a React component to be displayed inside the Tooltip.

<Canvas>
  <Story id="core-tooltip--custom-content" />
</Canvas>

### Placement

To position the Tooltip, use the `placement` prop. The component will flip and shift if there isn't enough space for it to be visible.

<Canvas>
  <Story id="core-tooltip--placement" />
</Canvas>

### Delay before shown

By default, the Tooltip displays after 300ms. To change the delay before it’s shown, use the `enterDelay` prop.

## Accessibility

The Tooltip doesn’t receive focus, so we recommend that you don’t include interactive elements as they won’t be accessible. If you need to display focusable content such as buttons or links, consider using the Overlay component instead.
