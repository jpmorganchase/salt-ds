# Checkbox

A Checkbox allows the user to turn a specific value on or off. The value can be an independent value or a selection of one or more values from a given set of choices (structured as a Checkbox Group).

##### Use when

- You want to present an independent choice that the user can toggle on or off.
- You want to present a list of independent options where the user can select any number of choices.

##### Don't use when

- The checkbox displays a mutually exclusive choice between two or more values, in that case a `Radio Button` component is more appropriate.

## Features

### Variations

A Checkbox can be in one of two states, `unchecked` or `checked`

<Canvas>
  <Story id="core-checkbox--default" />
</Canvas>

### Indeterminate

A Checkbox with the prop `indeterminate="true"` will display the checkbox with an indeterminate icon, this is a third state separate from the `unchecked` and `checked` states.
The most common use case is a nested checkbox group which shows a hierarchy of selections. If a group contains a mix of selected and unselected
children, the parent will show an indeterminate state.

<Canvas>
  <Story id="core-checkbox--indeterminate" />
</Canvas>

### Error

A Checkbox with the prop `error="true"` will cause the checkbox to be shown with the error status colors.
Use this state when you want to alert the user of a critical issue that is related to the checkbox.

<Canvas>
  <Story id="core-checkbox--error" />
</Canvas>

### Disabled

A Checkbox with the prop `disabled="true"` will suppress all functionality along with disabling the Checkbox.

</Canvas>

## Checkbox Group

A checkbox group allow selection of one or more values from a given set of choices. They can be aligned horizontally or vertically.

### Vertical group

<Canvas>
  <Story id="core-checkbox--uncontrolled-group" />
</Canvas>

### Horizontal group

The checkbox group layout can be set to horizontal by setting the `direction` prop to "horizontal"

<Canvas>
  <Story id="core-checkbox--horizontal-group" />
</Canvas>

### Wrapping

By default, the horizontal checkboxes will wrap onto the next line if the group doesn't fit within the container. There is the option to force the checkboxes to remain inline and wrap the label text instead by setting `wrap` prop to `false`.

`wrap=true`

<Canvas>
  <Story id="core-checkbox--wrap-group" />
</Canvas>

`wrap=false`

<Canvas>
  <Story id="core-checkbox--no-wrap-group" />
</Canvas>

### Controlled

A controlled checkbox group requires events to trigger the state change. The application must track state and update the props manually. This is useful for when logic or validation needs to be applied or a request should happen when updating a form.
